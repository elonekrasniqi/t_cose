Detailed Description

This hold keys so they can pass through t_cose to the underlying cryptographic library where they are used. It is used for all keys for all algorithms in the t_cose API whether they are symmetric, public or private.

To fill this in, the particular use, key type and algorithm expected must be know. Further, the cryptographic library and how it uses this structure for a particular key and algorithm must be known. Looking at the t_cose example code is probably most direct path. Also see the general discussion for t_cose_key.h.

Initializers are provided for some common or standard key serialization formats.

t_cose_key is initialized to 0 and/or NULL pointers if it is not holding a key. Crypto adapters for libraries should honor this if possible.
