92/*
93 * TODO: move support for this to the test signer object.
94 * This selects a signing test mode called _short_ _circuit_
95 * _signing_. This mode is useful when there is no signing key
96 * available, perhaps because it has not been provisioned or
97 * configured for the particular device. It may also be because the
98 * public key cryptographic functions have not been connected up in
99 * the cryptographic adaptation layer.
100 *
101 * It has no value for security at all. Data signed this way MUST NOT
102 * be trusted as anyone can sign like this.
103 *
104 * In this mode, the signature is the hash of that which would
105 * normally be signed by the public key algorithm. To make the
106 * signature the correct size for the particular algorithm, instances
107 * of the hash are concatenated to pad it out.
108 *
109 * This mode is very useful for testing because all the code except
110 * the actual signing algorithm is run exactly as it would if a proper
111 * signing algorithm was run. This can be used for end-end system
112 * testing all the way to a server or relying party, not just for
113 * testing device code as t_cose_sign1_verify() supports it too.
114 */
115#define T_COSE_OPT_SHORT_CIRCUIT_SIG 0x00002000
